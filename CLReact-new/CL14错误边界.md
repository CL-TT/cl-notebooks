# 错误边界

## 1. 错误边界解决什么样的问题

比如有一个组件树，**其中的一个子组件出现渲染错误，那么就会导致整个页面崩溃，如果想要这个错误不让整个页面崩溃，那么就可以使用错误边界**，错误边界是一个组件（类组件写的)



## 2. 错误边界的使用

```jsx
import React from 'react';

class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  // 这个方法里面一般都是改变hasError的状态
  // 运行时间点 渲染子组件的过程中，发生错误之后，在更新页面之前，也就是页面崩溃之前
  static getDerivedStateFromError(error) {
    return { hasError: true }; // 显示降级 UI
  }

  // 这个方法一般记录错误信息
  // 运行时间点 渲染子组件的过程中，发生错误，更新页面之后才会触发，因为运行时间靠后，所以一般都用做错误信息记录
  componentDidCatch(error, errorInfo) {
    // 可以在这里上报错误日志
    console.error('组件报错：', error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      // 出错显示的 UI
      return <div>页面出错了，刷新试试~</div>;
    }
    return this.props.children;
  }
}

export default ErrorBoundary;

<ErrorBoundary>
  <div></div>
</ErrorBoundary>
```



## 3. 细节

1. 某些情况下是不可以捕获到错误的
2. 错误边界组件自身发生的错误
3. 异步发生的错误
4. 事件中的错误

所以一般只能捕获到渲染时的同步错误
