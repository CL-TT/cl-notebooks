# 组件渲染性能优化

## 1.类组件的渲染优化

想要组件重新渲染，无非就是组件依赖的props和state发生变化，那么就会导致重新渲染

**解决的问题就是更改过后的值并没有发生变化，却导致组件重新渲染了**，就希望它不会重新渲染

1. shouldComponentUpdate

```jsx
class App extends React.component {
    // 返回false，表名不会重新渲染， 新旧进行比较，是一种浅比较，基于Object.is方法
    shouldComponentUpdate(nextProps, nextState){
        if(baseEqual(this.props, nextProps) && baseEqual(this.state, nextState)){
            return false;
        }

        return true;
    }
    render() {
        
    }
}



/**
* 浅比较，只比较第一层的属性
**/
function baseEqual(target1, target2){
    for(let prop in target1){
        if(!Object.is(target1[prop], target2[prop])){
            return false;
        }
    }
    
    return true;
}
```

2. PureComponent 纯组件

```jsx
PureComponent其实就是对shouldComponentUpdate的一种封装，就不需要你自己写这个方法了，当然它对props和state的比较也是浅比较

class App extends PureComponent {
    render() {
        
    }
}
```



## 2. 函数组件的渲染优化

函数组件的state因为使用的是hooks, 所以不会出现那个问题，hooks中做了处理，  如果是props的话

可以使用react.Memo, 它是一个高阶组件，它内部也是依托于纯组件来实现的

```jsx
// 就是组件依赖的props更改时，值没有发生变化，那么就不会重新渲染这个组件

function App() {
    return (
      <div></div>
    )
}

export default React.Memo(App)
```



```jsx
function Memo(Comp) {
    return class newComp extends PureComponent {
        render() {
            return <Comp { ...this.props } />
        }
    }
}
```



## 3. useCallback 缓存函数

useCallback会得到一个固定引用值的函数，也就是缓存函数

```js
它是解决这样的问题的，比如说有这样的一个场景，我把一个函数作为props传给了一个子组件，这个子组件的父组件因为某个state的值更改了导致父组件重新渲染，那么就会导致这个函数被重新创建，就会导致这个子组件也会重新渲染， 问题在于这个子组件并不需要重新渲染，那么就可以利用useCallback来缓存这个函数，那么下次比较的事后它的引用值是一样的，就不会重新渲染


// 第一个参数，传入函数
const callback = useCallback(() => {
    //处理逻辑
}, [依赖项] );
```



## 4. useMemo 缓存的是这个函数返回的值  （类似于vue里面的计算属性）

```jsx
import { useMemo, useState } from 'react'

function App() {
    const [ a ] = useState(10)
    const [ b ] = useState(10)
    
    const total = useMemo(() => {
        return a + b
    }, [a, b])
    
    return (<div></div>)
}
```
