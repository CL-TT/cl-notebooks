## 2. 配置文件package.json的介绍

1. 使用命令  npm init  或者  npm init -y | npm init --yes  这种方式配置项全部都默认

2. 配置文件中可以描述大量的信息，包括：

​    1. name：包的名称，该名称必须是**英文单词字符**，支持连接符

​    2. version：版本

​     **版本规范：主版本号.次版本号.补丁版本号**

​     主版本号：仅当程序发生了重大变化时才会增长，如新增了重要功能、新增了大量的API、技术架构发生了重大变化

​     次版本号：仅当程序发生了一些小变化时才会增长，如新增了一些小功能、新增了一些辅助型的API

​     补丁版本号：仅当解决了一些 bug 或 进行了一些局部优化时更新，如修复了某个函数的 bug、提升了某个函数的运行效率

​    3. description：包的描述

​    4 .homepage：官网地址

​    5. author：包的作者，必须是有效的 npm 账户名，书写规范是 ```account <mail>```，例如：```zhangsan <zhangsan@gmail.com>```，不正确的账号和邮箱可能导致发布包时失败

​    6. repository：包的仓储地址，通常指 git 或 svn 的地址，它是一个对象

​    7. type：仓储类型，git 或 svn

​    8. url：地址

​    9. main：包的入口文件，使用包的人默认从该入口文件导入包的内容

​    10. keywords: 搜索关键字，发布包后，可以通过该数组中的关键字搜索到包

3. 两个依赖

   开发依赖和生产依赖

1. **package.json文件最重要的作用，是记录当前工程的依赖**

​    1. dependencies：生产环境的依赖包

​    2. devDependencies：开发环境的依赖包

   2.在拿到别人的工程后如何下载node_modules

​     1. npm install  |  npm i  下载开发依赖和生产依赖 

​     2. npm install --production  仅下载生产时依赖  

3. 使用命令具体保存到哪个依赖中

​    \## 安装依赖到生产环境

​    npm i 包名

​    npm i --save 包名

​    npm i -S 包名

​    \## 安装依赖到开发环境

​    npm i --save-dev 包名

​    npm i -D 包名
