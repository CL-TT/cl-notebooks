# 包的使用以及导入规则

## 1. 包的使用

当使用nodejs导入模块的时候，如果模块路径不是以./或者../开头，则node会认为导入的模块来自于node_modules

```js
const _ = require('lodash')

//直接导入即可使用
```





## 2. 导入时寻找包的规则(适用公共模块和自有模块)      面试题

**注意node中的内置模块是不适用这个规则的**

```js
1. 比如说我要导入一个模块， lodash模块

如果是非相对路径

const _ = require('lodash')

2. 先会从当前文件夹的node_modules文件下去找，有没有lodash.js这个文件

3. 如果没有的话，再找有没有lodash这个文件夹， 去找他的入口文件(入口文件规则如下)

4. 如果有这个文件夹的话，就找该文件夹的package.json文件， 找main字段， 该字段的值就是入口文件

5. 如果没有没有入口文件或者没有配置文件， 就找index.js

6. 以上都没有，就到上层文件以这样的规则去寻找， 直到最上层文件


如果是相对路径
require('./xx')

1. 先从当前目录下找有没有这个文件

2. 如果没有就在package.json配置中找main这个字段，，看看在xx文件夹下面有没有这个字段对应的文件

3. 如果还没有的话，那么就找xx文件夹下面的index.js文件
```
