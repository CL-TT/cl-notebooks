# 支付页面

## 1. 微信支付

### 1.1 哪些账号可以实现微信支付

1. 企业账号
2. 企业账号的小程序后台，必须给开发者添加白名单， 这样一个APPID可以绑定多个开发者， 这些开发者就可以公用这个APPID和开发权限。

### 1.2 微信支付的流程

1. 点击支付按钮过后，先判断缓存中有没有token
2. 如果没有的话进行到授权页面获取token，  是要后端的接口的(所以这一步我没办法做)， 请求对应接口后端返回一个token值
3. 如果有的话，那么就可以创建订单， 同样这个创建订单也是要后端接口的(参数就是这个token值)， 获取订单编号
4. 发起预支付接口
5. 发起微信支付
6. 查询后台，订单的状态
7. 已经完成了微信支付
8. 就要手动的删除缓存中被选中的商品，因为已经付过钱了

## 2. 支付功能的实现

1. 因为没有接口，所以只能模拟去实现

2. 没有token值时，跳转到授权页面， 模拟一个token值，然后保存到缓存中

3. 有token值时， 然后模拟一个订单编号

4. 然后会询问是否确定支付， 如果确定， 然后把支付过的商品移除购物车

5. 然后把订单和一些商品数据保存到云数据库中， 形成订单列表数据库
