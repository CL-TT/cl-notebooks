# 正则深度进阶规则

## 1.捕获组

   \* 1.用小括号包裹的部分叫做捕获组，捕获组会出现在匹配结果中{

   \*  result = reg.exec(str)  result[1]就可以得到捕获的结果

   \* }

   \* 2.捕获组可以命名，叫做具名捕获组{

   \*  /(?<name>)/

   \*  在数组中有一个group,这个group是一个对象 保存着捕获的数据

   \* 

   \*  str.replace(reg, function(match, g1, g2, g3)) g1代表捕获组

   \*  str.replace(reg, '$1')  $1就代表着第一个捕获组

   \* }

   \* 3.又想用括号包裹起来但 又不想被捕获 可以用非捕获组{

   \*  /(?:)/  因为捕获会浪费效率的，当只想看成一个整体时就用非捕获组

   \* }

   \* 

## 2.反向引用

   \* 1.在正则表达式中，使用某个捕获组，```\捕获组编号```{

   \*  /(\d{2})\1/    就是匹配这样的数字 1212

   \* }

   \* 

## 3.正向断言(预查)

   \* 1.检查某个字符后面的字符是否满足某个规则，该规则不成为匹配结果，并且不称为捕获组{

   \*  1.我理解的正向断言就是对你所查找的东西 进行一定的条件限制， 但不会匹配到限制的结果

   \*  2./\d(?=\w+)/  (?=) 正向预查

   \*  3.每一次预查都是从头开始， 并不消耗字符

   \* }

   \* 

##  4.负向断言(预查)

   \* 1.检查某个字符后面的字符是否不满足某个规则，该规则不成为匹配结果，并且不称为捕获组 {

   \*  1.(?!) 负向预查

   \* }
