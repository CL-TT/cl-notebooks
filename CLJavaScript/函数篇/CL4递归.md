# 递归

## 1.递归和死循环的比较

1. 递归指的是函数直接或间接调用自身

   避免无限递归，无限递归会导致执行栈溢出，并报错。

2. 对比死循环

   1. 死循环不会报错，也不会导致栈溢出，如果在浏览器环境执行，会导致页面卡死。

   2. 无限递归会导致执行栈溢出



## 2.执行栈

任何代码的执行都必须有一个执行环境，执行环境为代码的执行提供支持，执行环境是放到执行栈中的。

每个函数的调用，都需要创建一个函数的执行环境，函数调用结束，执行环境销毁。

执行栈有相对固定的大小，如果执行环境太多，执行栈无法容纳，会报错

```javascript
function a(){
    console.log('a开始');
    function b(){
        console.log('b开始');
        console.log('b结束');
    }
    b();
    console.log('a结束');
}
a();

//先打印 a开始 ->   b开始   ->    b结束    ->    a结束
//这段代码的执行栈流程如下图
```

## 3.尾递归

如果一个函数最后一条语句是调用函数，并且调用函数不是表达式的一部分，则该语句称为尾调用，如果尾调用是调用自身函数，则称为尾递归。

```javascript
return f(n)     //像这样不是表达式的一部分， 并且只调用了自身函数，那么这就是尾递归。
```

某些语言或执行环境会对尾调用进行优化，它们会理解销毁当前函数，避免执行栈空间被占用。

在浏览器执行环境中，尾调用没有优化。但在nodejs环境中有优化。



## 4.递归的应用

### 4.1求阶乘

```javascript
//输入一个数字，求出这个数字的阶乘
function jc(n){
    if(n === 1){
        return 1
    }else{
        return n * jc(n - 1);
    }
}
```





### 4.2斐波拉切数列

```javascript
//斐波拉契数列是这样一种数列：1  1  2  3  5  8  13 ......
//数列前两位为1，第n位=第n-1位+第n-2位
//你想要输出第几位

function f(n){
    if(n === 1 || n === 2){
        return 1
    }else{
        return f(n - 1) + f(n - 2);
    }
}
```





### 4.3汉诺塔

```javascript
//A   B   C三根珠子    和n个汉诺塔     小塔放在大塔上面，大塔不能放在小塔上面,  要把所有的汉诺塔移动到C处
//n = 1 ,    A -> C           f(A, B, C, n = 1)
//n = 2 , A -> B  A -> C  B -> C    
//      f(A, C, B, 1) ->   A -> C   ->    f(B, A, C, 1)
//n = 3,  先把上面两个看成一个整体， 进行n = 2的操作 
//      f(A, C, B, 2) ->   A -> C   ->    f(B, A, C, 2)
//以此类推n = n
//  f(A, C, B, n - 1)   ->    A -> C     ->     f(B, A, C, n - 1)


function h(N1, N2, N3, n){
    if(n === 1){
        console.log(`${N1} -> ${N3}`);
    }else{
        h(N1, N3, N2, n - 1);
        console.log(`${N1} -> ${N3}`);
        h(N2, N1, N3, n - 1);
    }
}
```
