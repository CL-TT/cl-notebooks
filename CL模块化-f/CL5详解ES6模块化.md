# 详解ES6模块化

## 1. ES6模块化的诞生与特点

1. ECMA组织参考了众多社区模块化标准， 终于在2015年发布官方的模块化标准ES6模块化标准

2. ##### 具有以下特点

   2.1. **使用依赖预声明的方式**， 在文件的开头就声明引入哪些模块， 这种方式的**优点在一开始就可以确定模块的依赖关系**， 缺点是某些时候**效率低下**

   2.2. **commonjs使用的是依赖延迟声明**， 这种方式的优点是某些时候可以提高效率， 但缺点是无法在一开始确定模块依赖关系（比较模糊）

   2.3. 灵活的多种导入导出方式

   2.4. 规范的路径表示法， 所有路径必须使用./或者../开头



## 2. 基本导入导出

```js
1. 在使用script标签导入的时候， 需要加上 type = 'module' 这个标记，记录的是导入的是模块

2. 基本导入和导出的用法{
    1.文件地址只能以  ./ 或者 ../开头
    2.导入的模块数据不可更改
    
    导入方式{
      这些导入的模块最好写在代码的开头
      1. import  ''    只执行模块代码，不导出对象
      2. import { name, age } from ''  这个{}不是一个对象，只是一个列表符号
      3. import * as 别名 from ''  *是导出所有  as是帮变量起别名
    }
    
    导出方式{
      export let name = 'caolei'
      export function add(){}
      export {
         name,
         age
      }
      
      这个{}不是一个对象，只是一个列表符号
}
```





## 3. 默认的导入导出

```js
默认导入导出{
   1. 导出{
        export default 可以直接写数据
        默认导出只能写一次
      }
   
   2. 导入{
        import data from ''
        这个data变量就是接受导出的数据的
      }
}
```





## 4. require和import的区别？  面试题

1. ##### 规范不同

   require是CommonJs/AMD规范

   import是ES6模块化规范

2. ##### 调用时间不同

   require是运行时调用，所以require可以运用在代码的任何地方

   import是编译时调用，所以必须放在文件开头
