# 详解CommonJs

## 1. 什么是模块？

1. **模块就是一个js文件， 它实现了一部分的功能， 并隐藏自己的内部实现， 同时提供了一些接口供其他模块使用**
2. 模块有两个核心， **隐藏与暴露**， 隐藏自己内部的实现， 暴露外部想用的接口



## 2. CommonJs的规范

1. CommonJs使用exports， module.exports导出模块,  使用require导入模块

#### 2.1 具体规范

1. 如果一个JS文件中存在```exports```或```require```，该JS文件是一个模块
2. 模块内的所有代码均为隐藏代码，包括全局变量、全局函数，这些全局的内容均不应该对全局变量造成任何污染
3. 如果一个模块需要暴露一些API提供给外部使用，需要通过```exports```导出，```exports```是一个空的对象，你可以为该对象添加任何需要导出的内容
4. 如果一个模块需要导入其他模块，通过```require```实现，```require```是一个函数，传入模块的路径即可返回该模块导出的整个内容



#### 2.2 CommonJs的特点

1. 它是社区标准，并非官方标准
2. 使用函数实现的
3. 仅在node环境下运行
4. 动态依赖，在代码运行后才能确定依赖
5. 动态依赖是同步的



## 3. nodeJs如何实现CommonJs的？   面试题

1. 为了保证高效的执行， 仅加载必要的模块， nodejs只有执行到require函数时才会加载并执行模块
2. 为了隐藏模块中的代码，nodejs执行模块时，会将模块中所有的代码放置到一个函数中执行，以保证不会污染全局变量
3. 为了保证顺利导出， nodejs做了以下处理
4. **commonjs是同步的**

```js
(function(module){
    module.exports = {
        
    };
    
    let exports = module.exports;
    
    //模块中的代码
    
    // 记住导出的是module.exports
    return module.exports;
})()
```

4. 并且为了避免重复加载同一个模块， nodejs默认开启模块缓存的方式， 如果加载的模块已经被加载过了，则会自动使用之前的导出结果



```js
const b1 = require('./b');

const b2 = require('./b');

console.log(b1 === b2)  // true    因为有缓存，所以b1是等于b2的
```





## 4. 为什么CommonJs模块化规范在浏览器端使用不了？   面试题

#### 1. 弄清楚在node环境， commonjs是如何运作的

1. 通过模块的路径找到本机文件，并读取文件内容
2. 将文件中的代码放到一个函数环境中执行，并将执行后的module.exports的值作为require函数的返回结果

#### 2. 当commonjs放到浏览器端遇到的挑战

```html
// 入口的js文件
<script src="./index.js"></script>


// 在index.js文件中
require('./a.js')
```

1. 在加载完入口文件后， **要加载js文件， 就需要从远程的服务器中读取， 而网络传输的效率要远远低于node环境读取本地文件的效率， 并且commonjs是同步的，这会极大的降低运行性能**

2. 就算读取到了js文件的内容， **但是需要把它放入到一个环境中去执行， 这是需要浏览器厂商支持的， 但是由于commonjs只是社区的规范， 并非官方的标准， 都不愿意提供支持**



#### 3. 重新制定规范适应浏览器的模块化

1. 在这个大背景下AMD， CMD应运而生
2. 解决办法其实很简单：

   1. 远程加载JS浪费了时间？做成异步即可，加载完成后调用一个回调就行了
   2. 模块中的代码需要放置到函数中执行？编写模块时，直接放函数中就行了
   3. 基于这种简单有效的思路，出现了AMD和CMD规范，有效的解决了浏览器模块化的问题。
