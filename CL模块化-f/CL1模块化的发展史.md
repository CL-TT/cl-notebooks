# 模块化的发展史

## 1. 第一阶段（js语言的诞生）

1. javascript语言诞生的时候，仅仅用于实现页面中的小效果（被戏称为玩具语言）

#### 1. 发生的大事

1. 1996年 网景公司将javascript语言提交给欧洲的一个标准制定组织ECMA
2. 1998年 网景在和微软的IE浏览器竞争中失利，宣布破产



## 2. 第二阶段 (ajax把js带向了新的高度)

1. ajax的出现，逐渐改变了javascript在浏览器中扮演的角色， 它不仅能实现小的效果，还可以和服务器之间进行交互
2. 随之带来的就是前端程序逐渐变得复杂

#### 1. 前端开发也出现了难以突破的瓶颈

1. 浏览器解释执行js的速度太慢
2. 用户端的电脑配置不足
3. 更多的代码带来了全局变量污染，依赖关系混乱等问题

#### 2. 发生的大事

1. IE浏览器制霸市场， 几乎不再更新
2. ES4.0流产， 导致js语言将近10年几乎毫无变化
3. 2008年发布的ES5，也只是解决了一些JSAPI不足的糟糕局面



## 3. 第三阶段 （v8 => nodejs => commonjs）

1. 2008年**谷歌的v8引擎发布，将js的执行速度推向了一个新的台阶**，甚至可以和后端语言媲美
2. 摩尔定律持续发酵，解决电脑配置不足的局面
3. 至此，制约前端发展的最后一个问题， 即**全局变量污染和依赖关系混乱**解决了它前端便可以突破一切障碍，未来有无限的可能
4. 2009年，**Ryan推出了nodejs，从此，JS第一次堂堂正正的入主后端**，不再是必须附属于浏览器的“玩具”语言了。
5. nodejs的诞生，便把js中的最后一个问题放到了台前， 即**全局变量污染和依赖混乱**问题， 必须要解决了
6. 所以CommonJs模块化方案，孕育而生， **nodejs成为了第一个为js语言实现模块化的平台**

#### 1. 发生的大事

1. 2008年， v8发布
2. IE市场逐渐被火狐，谷歌所蚕食， 已无力回天
3. 2009年，nodejs发布， 并附带commonjs模块化标准



## 4. 第四阶段 （ES6模块化）

1. 人们想办法，想把commonjs运用到浏览器中（可是存在这很多问题，最终没能实现）
2. 所以人们想制定一个适合浏览器的模块化标准， **AMD， CMD, 应运而生**
3. 2015年， ES6发布，**他提出了官方的模块化解决方案 ES6模块化**， 至此，js这门语言有了无穷的潜力
4. js也能开发大型应用 => 三大框架出现  =>  express, koa后端开发框架出现， 各种后端数据库驱动出现， =>  开发大型应用，少不了第三方库的支持，npm等包管理器出现， webpack等构建工具出现
