# EffectHook处理副作用钩子

## 1.Effect Hook的作用

1. **Effect Hook用于在函数组件中处理副作用**



## 2.什么是副作用

1. ajax请求
2. 计时器
3. 其他异步的操作
4. 更改真实的Dom对象
5. 本地存储
6. 其他会对外部产生影响的操作



## 3.使用useEffect

1. useEffect是一个函数，第一个参数可以传递一个函数，处理逻辑都在这个函数里面

   

2. ```react
   useEffect(() => {
       
   }, []);
   
   //空数组的目的是让该副作用函数没有任何依赖项，从而仅在首次挂载时运行
   
   //依赖项发生变化，就会重新执行
   ```

   

3. 这个副作用函数的运行时间点，**是在页面已经渲染完成，人已经可以看到真实的ui界面之后，会执行这个副作用函数，这个函数的执行是异步的，并且不会阻塞浏览器**

   

4. 与类组件中的componentDidMount和componentDidUpdate的区别

   1.componentDidMount和componentDidUpdate的运行时间点是更改了真实的Dom之后就会运行，但是用户还没有看到ui更新，这是同步的，所以说会阻塞浏览器

   2.useEffect中的副作用函数，更改了真实的Dom，并且用户已经看到了ui更新，异步的，不会阻塞浏览器

   

5. 每个函数组件中可以多次使用副作用操作，但是不要放入判断，循环的代码块中

   

6. useEffect中的副作用函数，可以有返回值，返回值必须是一个函数，该函数叫做清理函数

   1.**清理函数的运行时间点， 在每次运行副作用函数之前**，**首次渲染组件不会运行**

   2.组件被销毁时一定会运行清理函数，但是组件被销毁时不会运行副作用函数
