# 数组和链表的区别

1. 数组和链表是两种基本的数据结构， 他们在内存存储上的表现不一样，所以各自有各自的特点

比如说，拿几个人去看电影坐座位为例

## 1. 数组

1. 在内存中，**数组是一块连续的区域**，这几个人在电影院必须坐在一起
2. **数组需要预留空间， 在使用前要先申请占内存的大小**，可能会浪费内存空间， 比如说看电影，为了保证5个人能坐在一起， 就必须提前订好5个连续的位置， 这样的好处能保证5个人在一起，但缺点也很明显， 如果来的人不够5个那么剩下的位置就浪费了， 如果来的人超过了5个，那么位置就不够用就没法坐了
3. **插入数据和删除数据的效率很低**， 插入数据的时候，这个位置后面的数据在内存中都要向后偏移， 删除数据的时候，这个数据后面的 数据在内存中都要向前偏移
4. **随机读取的效率很高**， 因为数组是连续的，知道每一个数据的内存地址，可以直接找到给地址的数据
5. **不利于扩展**， 数组定义的空间不够时就要重新定义数组



## 2. 链表  (value ,  next: 下一个数据的地址)

1. **在内存中可以存在任何地方， 不要求连续**， 在电影院中几个人可以随便做
2. 每一个数据都保存了下一个数据的地址， 通过这个地址找到下一个数据， 第一个人知道第二个人的座位号， 第二个人知道第三个人的， 以此类推
3. **增加数据和删除数据是很容易的**， 再来一个人可以随便做， 其他人都不用动
4. **查找数据效率低， 不具有随机访问性**， 访问某个位置的数据都要从第一个数据开始访问
5. **不指定大小，扩展方便**， 链表大小不用定义， 数据随意删减
