# 为什么0.1 + 0.2 等于 0.30000000000000004

## 1. js中小数或者整数的运算是精确的吗？

可能是精确的，也可能不是

```
0.1 + 0.2 = 0.300000000000004

0.5 + 0.5 = 1
```



## 2. 十进制和二进制的转换

```js
1. 1101转换成10进制

1 * 2^3 + 1 * 2^2 + 0 * 2^1 + 1 * 2^0 = 8 + 4 + 0 + 1 = 13


2. 1101.01转换成10进制
整数部分：13
小数部分：0 * 2^-1 + 1 * 2^-2 = 0 + 0.25 = 0.25


3. 13转成二进制

整数部分从下往上读  1101

13 / 2  6  余 1
6 / 2   3  余 0
3 / 2   1  余 1
1 / 2   0  余 1


4. 13.25转成二进制

小数部分，从上往下读，一直到小数部分为0停止  01
0.25 * 2  0.5   0
0.5 * 2   1.0   1
```





## 3. 为什么js的小数运算不精确

1. **十进制的小数，转换成二进制后，可能是无限小数，但是计算机对数字的存储能力是有限的，因此会丢掉一些数据**



## 4. js如何存储数字的

1. 有两种方法， 整数法， 和浮点法
2. **js中存储的所有数字，都按照浮点法存放**，
3. 浮点法存放的数字叫做浮点数（float），浮点数分为单精度和双精度
4. **js中使用双精度存放浮点数，IEEE 754**

1 byte(字节) = 8 bit
1 kB = 1024 byte
