# 如何实现私有属性

## 1. 利用约定俗成的方式

双方约定好以_开头或者什么符号开头的属性是私有属性，这种方式是弱约束，如果要访问的话还是能访问到的

```js
const obj = {
    name: 'caolei',
    _age: 20
}
```



## 2. Symbol符号

符号的设计初衷就是为了给对象设计私有属性

```js
const key = Symbol('key')

const obj = {
    name: 'caolei',
    [key]: 'hello world'
}

直接通过obj是访问不到的，遍历也访问不到

但还是有办法能访问的，创建的Symbol如果能访问到的话 obj[key]

一般创建的时候封装成一个模块不导出符号只导出类，让别人看不到， 但还是可以通过 Object.getOwnPropertySymbols(obj)获取到这个对象的所有Symbol属性
```



## 3. TS环境下的关键字private

```ts
class A {
    private name = 'caolei'
}

const a = new A()

也是有缺陷的，直接a.name是会报错，但是可以换一种形式啊a['name'] 动态的属性
关键的是ts它是编译时态的产物， 在编译结果里面是不存在这个关键字的， 如果要强行访问的话也是可以访问的， 
```



## 4. 原生语言里面的私有变量语法

```js
class A {
    name = 'caolei'
    #age = 20
}

const a = new A()

这种方式是完全访问不到这个属性的，是万无一失的， 但也有自己的缺陷，就是它是新语法，为了做兼容处理，可能还不能使用这种方式
```



## 5. 终极方案

```js
不在这个类里面写字段，把这些字段放入到map集合里面， (属于一种降级方案)

const filedMap = new WeakMap()

class A {
    constructor() {
        // 把这些私有属性映射到集合中
        filedMap.set(this, {
            key: 'caolei'
        })
    }
    
    print() {
        console.log(filedMap.get(this).key)
    }
}
```
