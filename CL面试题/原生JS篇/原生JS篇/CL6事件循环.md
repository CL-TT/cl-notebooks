# 事件循环原理

## 1.五大线程

1. js引擎，js主线程
2. GUI线程(渲染页面的线程)
3. 事件监听线程
4. 定时器线程
5. 网络线程



## 2.事件循环流程

1. 正常解析js代码

2. 分析是同步任务，还是异步任务

3. 如果是同步任务，就走js主线程，有一个执行栈，把这个任务的执行期上下文放入执行栈中，入栈操作

   执行完后就销毁自己的执行期上下文

4. 如果是异步任务，在事件表(EventTable)中注册回调函数，如果外界触发了这个回调函数，那么就会把这个回调函数放入事件队列(EventQueue)中

5. 如果执行栈中的任务全部执行完了，那么就会到事件队列中一个个的取出回调函数放入执行栈中，事件队列分为微队列和宏队列，先取微队列再取宏队列，然后取出的回调函数会产生自己的执行期上下文，执行完销毁

   **宏队列：计时器结束的回调， 事件回调， http回调等绝大多数异步函数进入宏队列**

   **微队列：Promise产生的回调会进入微队列， MutationObserver是监听某个Dom对象的变化**

6. 这样就形成了事件循环操作
