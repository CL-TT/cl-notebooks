##### 1. http各版本的差异

```js
*1. http1.0

#1. http1.0的特点

1.它的特点是每一次请求和响应完毕之后都会销毁TCP连接，同时规定前一个响应完成之后才能发送下一个请求

#2. http1.0的两个问题

1. 【无法复用连接】， 每一次请求都要创建新的TCP连接，都要完成三次握手和四次挥手，就会导致无法充分的利用带宽，造成带宽资源的浪费，这种原因就是因为TCP协议的特点是【慢启动】即一开始传输的数据量少。一段时间之后达到传输的峰值，而上面这种做法就会导致大量的请求在TCP达到传输峰值前就被销毁了

2. 【队头阻塞】， 如果前一个请求被某种原因阻塞了，会导致后续请求无法发送
```

```
*2. http1.1   它是http1.0的改进版

1. 【长连接】

http1.1 允许在请求时增加请求头connection:keep-alive，这样便允许后续的客户端请求在一段时间内复用之前的TCP连接

2. 【管道化】

基于长连接的基础，管道化可以不等第一个请求响应完成，便可以继续发送后面的请求，但响应的顺序还是按照请求的顺序返回，所以还是有队头阻塞的问题存在， 它并不是一个成功的模型，它带来的队头阻塞会引起其他的问题，所以现代浏览器默认关闭这种模式

3. 【缓存处理】

新增响应头 cache-control 用于实现客户端缓存

4. 【断点传输】

在上传或者下载资源时，如果资源过大，将其分割为多个部分，分别上传下载，如果遇到网络故障，可以从已经上传下载好的地方继续请求，不用从头开始，提高效率


一般优化队头阻塞的方法：
1. 减少文件的数量
2. 通过开辟多个TCP连接


```

```
*3. http2.0

1. 【二进制分帧】

将传输的消息分为更小的二进制帧，每帧都有自己的标识序号，即使被随意打乱也能在另一端正确组装， 基于二进制分帧，不再有队头阻塞问题，无需遵守响应顺序

比如说，现在有两个请求，请求a，请求b, 服务器作出响应把每一个响应都分为2帧， 每一帧都包含如下内容， 分为帧的头部和帧的内容，帧的头部又分为所属的流，是属于哪一个请求的，类型，是响应头还是响应体

2. 【头部压缩】

在之前所有的消息头都是以字符的形式完整的传输的，实际上大部分头部信息都有很多重复的，http2.0 通过字典的形式，将头部常见的信息替换为更少的字符，极大的减少了头部的数据量，从而实现更小的传输量

3. 【服务器推送】

http2.0 允许服务器直接推送消息给客户端，无须客户端明确的请求
```



##### 2. 浏览器缓存

```js
理念：浏览器发出请求，先会从缓存中去看一下有没有这个请求，因为这是第一次发送缓存中肯定是没有的，服务器接收到请求响应了资源给浏览器，然后把资源保存到缓存中，第二次请求缓存中已经有了直接返回数据，这个请求就不需要再发送了

1. 【按缓存位置分类】

从缓存位置上来讲，可以分为四种，并且各自有优先级，当依次查找缓存且都没有命中的时候，才会去请求网络

#1. Service Worker

它是运行在浏览器背后的独立线程可以用来实现缓存功能

#2. Memory Cache

Memory Cache也就是内存中的缓存，主要包含页面中已经抓取到的资源，比如样式，脚本，图片等

#3. Disk Cache

Disk Cache也就是存储在硬盘中的缓存，读取速度慢一点，但是在容量以及存储时效上要比Memory Cache更胜一筹

#4. Push Cache

它是推送缓存，当以上三种缓存都没有命中时它才会使用，它只在会话（Session）中存在，一旦会话结束就会被释放，并且缓存时间也很短暂


2. 【按缓存类型分类】

按照缓存类型可以分为强制缓存和协商缓存，无论是哪一种都是属于Disk Cache或者叫Http Cache

强制缓存：
1. http1.0中的Expires 绝对时间
2. http1.1中的cache-control: max-age  相对时间

协商缓存：
1. Last-Modified / if-Modified-Since
2. Etag / if-None-Match

过程如下

1. 浏览器发出请求，服务器有一段资源想让浏览器缓存起来，那么就需要标识告知浏览器这段资源需要缓存, 那么响应头就会有如下字段告知浏览器

Cache-Control: max-age   过期时间
Etag: 资源编号
Date: 获取资源的时间
Last-Modified: 上一次修改资源的时间

2. 浏览器得到请求资源之后缓存起来

3. 下一次再发送请求，那么就会从缓存中去找，如果缓存中不存在该请求，那么就发送正常的请求， 如果缓存中存在， 看缓存的时间有没有过期

4. 如果没有过期那么就直接走缓存就行了，不需要发送请求， 如果过期了，那么就向服务器发送一个带缓存标识的请求(协商缓存)，看服务器的资源有没有改变，请求头的字段如下

if-Modified-Since: 上一次修改资源的时间
if-None-Match: 资源编号

两个字段是为了兼容，有些浏览器认第一个，有些认第二个，有些两个都认，   服务器根据这两个字段去比较，看资源有没有更新

5. 如果服务器的资源更新了，那么就重新响应一个状态码为200的，然后更新一下响应头相关的字段，  如果没有更新，  那么就响应一个状态码为304表示资源没有更新，并没有返回请求体，还可以继续使用缓存，只不过把缓存的时间重新更新一下， 然后继续拿缓存里面的内容
```
