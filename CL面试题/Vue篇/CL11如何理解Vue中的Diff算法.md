# 如何理解Vue中的Diff算法

## 1. 什么叫Diff算法

1. 新的虚拟Dom树和旧的虚拟Dom树进行比较，找到差异，最终更新真实Dom，对比差异的这个过程就做Diff, vue在内部通过一个叫patch的函数完成该过程。



## 2. 什么时候会发生Diff

1. 当组件创建时，以及依赖的属性或数据发生改变时，会运行一个函数，该函数会做两件事情

   1.1.  运行_render生成一个新的虚拟Dom树（vnode tree）

   1.2.  运行_update, 传入虚拟的Dom树的根节点，对新旧两棵树进行对比，最终完成真实Dom的更新

   1.3.  Diff就发生在_update函数的运行中

```javascript
//Vue构造函数
function Vue(){
    //会运行这个函数
    const updateComponent = () => {
        this._update(this._render());
    }
    
    new Watcher(updateComponent);
}

1. 由此可以看见交给watcher运行的也不是render函数， 创建或当有属性和数据发生改变，把updateComponent交给watcher去执行， 让用到的属性去收集依赖(也就是收集watcher)，当数据改变调用watcher，然后再次执行updateComponent函数，从而更新数据。
```



## 3. _update函数在干什么

1. _update函数接受一个参数，这个参数就是新生成的虚拟Dom树

```javascript
function _update(vnode){
    const oldVnode = this._vnode;  //先把旧的虚拟Dom树存起来
    this._vnode = vnode;     //新的虚拟Dom节点赋值给this._vnode，这一步虚拟Dom已经更新完毕

    if(!oldVnode){
        //如果是第一次渲染，就不存在旧树，那么就直接遍历新的虚拟Dom树
        //那么就会直接调用__patch__传入一个当前绑定的元素， 和新的虚拟Dom树，然后遍历新的虚拟Dom生成真实的Dom, 然后挂载到每个节点的elm属性上，真实的Dom的根节点挂载到el上
        this.__patch__(this.$el, vnode)
    }else{
        //如果这不是第一次渲染，还是要通过patch函数，来做新旧两个树的比较， 这个地方才有diff算法的过程
    }
}
```

## 4. patch函数的比较过程

```vue
概念术语：

1. 【相同】：是指两个虚拟节点的标签类型相同，并且key值相同，但input元素还要看type属性

<div>你好呀</div>
<div>我不好</div>
{ tag: 'div': key: undefined }
{ tag: 'div': key: undefined }

这两个虚拟节点的标签类型是完全相同的，key值也是相同的，所以他们是相同的


2. 【新建元素】： 是根据一个虚拟节点的信息，创建一个真实的Dom元素，同时挂载到虚拟节点的elm属性上

3. 【销毁元素】： 是指vnode.elm.remove()

4. 【更新】： 是指对两个虚拟节点进行对比更新，他仅发生在两个虚拟节点【相同】的情况下

5. 【对比子节点】： 是指对两个虚拟节点的子节点进行对比

```

![update3](D:\CL武林秘籍\画图\update3.png)

```vue
详细流程：

1. 对比根节点的过程

@1. 如果两个根节点【相同】 对比上述术语的相同概念， 进入【更新】流程
 #1. 将旧结点的真实Dom赋值到新节点的elm属性上   newVnode.elm = oldVnode.elm
 #2. 对比新节点和旧节点的属性，有变化的更新到真实的Dom中
 #3. 当前的两个节点处理完毕，开始【对比子节点】

@2. 如果两个根节点不相同
 #1. 新节点递归 【新建元素】
 #2. 旧结点 【销毁元素】


2. 对比子节点的过程 【理念和原则就是，尽量啥都不做，不行的话，尽量仅改变元素的属性，还不行，就尽量去移动元素而不是删除和创建元素， 还不行的话只能去删除或者创建元素了】， 过程如下

@1. 新旧子节点分别有一个头指针和一个尾指针，【先从头指针开始比较】，看两个节点是否【相同】，如果相同还是会走父节点相同的过程

@2. 如果不同，先什么都不做，【再看尾指针对应的两个节点是否相同】， 相同就更新

@3. 如果还是不同【交叉对比指针对应的子节点】，

@4. 如果交叉都没有相同， 那么就看这个新节点，在所有的旧节点中存不存在， 存在的话就更新，当然这其中包含着一些真实Dom的移动

```

## 5. 在实际开发中的运用

1. v-for循环为什么要加key值？有一个列表，经过翻转的操作

   如果你不加key值的话，他在比较的时候会认为两个子节点是相同的，就会走更新的过程，

   如果这个列表里面的内容非常多，改动的地方就非常多
   
   如果你加了key值，就会走指针的方式，这样只会去移动元素，而不会去更改元素
