# 跨域的几种方法详解

## 1. 为什么有跨域请求

因为浏览器有一个安全机制叫做同源策略，也就是协议域名端口号要一模一样，访问其他网页出现不同源情况时浏览器会进行数据拦截，所以需要跨域请求数据。

## 2. 哪些资源算跨域请求

1. 后端接口的数据
2. 其他域的cookie
3. 其他域的缓存

## 3. 怎样才算跨域

协议， 域名， 端口号 这三个任意一个不一样都算跨域

## 4. 跨域这个行为发生在哪里’

在浏览器接收到请求回来的数据时进行处理时，因为同源策略数据被拦截，这个时候才知道发生了跨域。

## 5. 解决跨域的方法

### 5.1 后端配合我们进行跨域

#### 5.1.1 jsonp

```javascript
//jsonp的原理
//1. 主要是借用script标签， 因为script标签的src是支持跨域的
//******前端部分*******
//2. 判断是否同源， 如果同源就用正常的ajax请求， 不是同源则动态生成一个script标签
//3. 生成一个随机的callback名字
//4. 设置script标签的src属性为要请求的接口地址
//5. 将callback作为参数拼接在url的后面
//*******后端部分******
//6. 后端接收到数据后，开始要准备返回的数据
//7. 后端拼接数据，用callback的值和括号包裹起来例，  callback(data)
// callback = asd123  =>   asd123({name:'caolei'}) => 这就是拼接后的数据
//8. 将内容返回，浏览器接收到会当做js代码来执行，从而执行了asd123的方法， 就得到了数据

jsonp跨域的缺点
1. 会打乱服务器的消息格式， jsonp要求服务器响应一段js代码，但在非跨域的情况下，服务器又要响应一个正常的json格式
2. 只能响应get请求的跨域问题，因为script标签只能发送get请求，

var $ = {
    ajax: function(options){
        var targetUrl = options.url,  
            targetDataType = options.dataType,
            targetProtocol = '',   //协议
            targetHost = '';   //域名和端口号
        
        if(targetUrl.indexOf('http://') == 0 || targetUrl.indexOf('https://') == 0){
            //包含这两个就算是跨域请求了
            var url = new URL(targetUrl);
            targetProtocol = url.protocol;
            targetHost = url.host;
        }else{
            targetProtocol = location.protocol;
            targetHost = location.host;
        }
        
        //判断dataType类型
        if(targetDataType == 'jsonp'){
            //如果是jsonp,那么判断是否同源
            if(targetProtocol == location.protocol && targetHost == location.host){
                //如果是同源，就进行正常的ajax请求
                return '';
            }else{
                //不是同源
                //动态创建一个script标签
                var script = document.createElement('script'),
                    //随机生成一个callback
                    callback = 'cb' + Math.floor(Math.random() * 100000000);
                    window[callback] = options.success;  //让全局有这个方法
                //判断有没有参数
                if(targetUrl.indexOf('?') != -1){
                    //有参数
                    script.src = targetUrl + '&callback=' + callback;
                }else{
                    //没有参数
                    script.src = targetUrl + '?callback=' + callback;
                }
                script.id = callback;
                //把标签添加到页面中
                document.head.appendChild(script);
            }
        }
    }
}
```

#### 5.1.2  后端设置 CORS (Corss origin resource sharing)  跨域资源共享

##### 1. 简单请求

请求方式只能为： get / post / head(不接受响应的数据，只拿到响应头的信息)

请求头当中的字段： Access   

​                                   Access-Language

​                                   Content-Type:  application/x-www-form-urlencoded

​                                                              mutipart/form-data

​                                                              text/plain

必须要设置 Access-Control-Allow-Origin : '*'  这个属性

##### 2. 不满足简单请求条件的请求

如果请求方式不一样： 那么就设置   Access-Control-Allow-Methods: 'put'

如果Content-Type不一样了 ： 那么就设置  Access-Control-Allow-Headers : 'Content-Type'

### 5.2 后端不配合我们进行跨域

#### 5.2.1 通过服务器代理

浏览器找到同一个端口下的服务器，利用这个服务器去请求我们需要跨域访问的那个服务器，因为服务器之间是没有同源策略的，这就是服务器代理。



#### 5.2.2 客户端之间的跨域， 父子窗口的跨域

##### 1.  iframe标签

父访问子： iframe + window.name

在子窗口中设置 window.name = JSON.stringify(obj)   //转换成字符串传递

在父窗口中

```javascript
var iframe = document.getElementById('');    //获取到iframe标签
var lock = false;
//必须等它加载完毕

    if(lock){
        return false;
    }
    lock = true;
    iframe.src = './index.html'   //改成本地的地址
    console.log(iframe.contentWindow.name);
}
```
