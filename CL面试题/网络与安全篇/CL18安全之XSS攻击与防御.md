# XSS攻击与防御

XSS：Cross Site  Scripting    跨站脚本攻击

## 1.存储型XSS

1. 恶意用户提交了恶意的内容到服务器
2. 服务器没有识别，保存了恶意内容到数据库
3. 正常用户访问服务器
4. 服务器在不知情的情况下，给予了之前的恶意内容，让正常用户遭受攻击

```js
解决方案 ， 主要就是防止用户写了恶意的脚本

npm i xss

const xss = require('xss');

//主要把script标签或者说把js代码去掉
const html = xss('');
```



## 2.反射型

1. 恶意用户分享了一个正常网站的链接， 链接中带有恶意的内容
2. 正常用户点击了该链接
3. 服务器在不知情的情况下，把链接的恶意内容读取出来，放进了页面中，让正常用户遭受到攻击



## 3.DOM型

1. 恶意用户通过任何的方式，向服务器注入了一些dom元素，从而影响了服务器页面中的dom结构
2. 正常用户访问时，运行的是服务器的正常js代码，但因为改变了dom结构，所以就会出问题

在模板引擎中渲染时，全部使用编码的形式渲染



## 4.防御

1. 过滤， 去掉一些危险的标签，和一些危险的属性
2. 编码： 对危险的标签进行HTML实体编码
