# JWT的使用（JSON web Token）

## 1. 为什么会出现Jwt

1. 随着前后端的分离，以及数据中心的建立，越来越多的公司会创建一个中心服务器，服务各种产品线

   这些产品线上的产品，可能有着各种各样的终端设备，包括但不仅限于浏览器，可能还会有移动端应用，桌面应用， 平板， 智能家居等

2. 实际上，不同的产品线通常有自己的服务器，产品内部的数据一般和自己的服务器进行交互，但中心服务器仍然有存在的必要，因为同一家的公司的产品总是会存在共享的数据，比如用户数据，这些设备与中心服务器之间会进行http通信

3. 一般来说，**中心服务器至少承担着认证和授权的功能，例如登录：各种设备发送信息到中心服务器，然后中心服务器响应一个身份令牌，这种结构就会出现一个问题，他们之间还能用传统的cookie传递令牌信息吗**

4. 其实是可以的。因为cookie在传输中无非是一个消息头罢了，只不过浏览器对这个消息头有特殊的处理，**但浏览器之外的设备肯定不喜欢cookie，因为浏览器对cookie有着完善的管理机制，但是在其他设备上就需要开发者手动的去处理了，所以Jwt的出现就是解决这个问题的。**

   

## 2. Jwt的特点

1. **他要解决的问题就是为各种终端设备，提供统一的，安全的令牌格式**
2. **对于传输，可以用任何的形式来传输jwt**, 但一般会用消息头来传输他

浏览器发送请求     =>         服务器

浏览器       <= jwt令牌           服务器

浏览器       =>携带jwt令牌      服务器

Get /api/student HTTP/1.1

authorization: bearer jwt令牌



浏览器         =>            服务器(验证jwt令牌)



### 2.1 jwt令牌的格式

```js
1. jwt令牌由三个部分组成
header: 令牌头部， 记录了整个令牌的类型和签名算法
payload: 令牌负荷  记录了保存的主体信息
signature: 令牌签名 按照头部固定的签名算法对整个令牌进行签名，签名的作用是为了保证令牌不被篡改和伪造

*2. header, 令牌的头部记录了整个令牌的类型和签名算法
{
    "alg": "HS256",
    "typ": "JWT"
}

alg: signature部分使用的签名算法， 通常可以取两个值
HS256： 一种对称加密算法，同一个秘钥对signature进行加密和解密
RS256： 一种非对称的加密算法，使用私钥加密，公钥解密

typ: 整个令牌的类型，固定写JWT即可

header部分就算生成了，然后用base64 url 编码即可

浏览器中的window.btoa(JSON.stringify({}))就可以做到转换


*3. payload   jwt的主体内容
{
    "ss" : "发行者名称",
    "iat": "发布时间",
    "exp": "到期时间",
    "sub": "主题",
    "aud": "听众发给哪个终端的",
    "nbf": "在此之前不可用",
    "jti": "JWT ID"
    
    "自己想要添加的属性"
}
这些属性全部都不是必填属性，他只是一个规范
同样需要转为base64 url编码


*4. signature  jwt签名,  保证jwt不被篡改和伪造
这部分的生成 是前面两个部分编码的结果作为加密字符串， header的算法作为加密算法， 然后你需要指定一个秘钥

HS256('前面两个部分编码的结果', '秘钥');

他为什么没有办法伪造？
因为jwt的最后一部分是被加密了的， 秘钥在服务器端，他拿不到秘钥就没有办法就行伪造

他为什么没有办法篡改？
他如果篡改了jwt的第二部分内容，然后发给了服务端，那么服务端会再用算法和秘钥对传过来的jwt的前部分进行加密，在和传过来的第三部分进行比较，  如果被篡改了比较肯定不会通过，服务器也就不会给客户端响应数据

```



## 3.Jwt的使用

1. npm i jsonwebtoken
2. const jwt = require('jsonwebtoken');
3. 颁发jwt   jwt.sign(payload, secret)
4. 验证jwt   jwt.verify(token)

## 4. 总结

1. JWT本质就是一种令牌格式，他和终端设备无关，同样和服务器无关，设置于如何传输无关，只是规范了令牌的格式而已
2. JWT由三部分组成， header   payload    signature
3. JWT难以被篡改和伪造，就是因为有第三部分的签名存在
