# HTTP1.0和HTTP2.0的区别

1991  HTTP/0.9

1996  HTTP/1.0

1999  HTTP/1.1

2015  HTTP/2.0

## 1. HTTP的历史

1. 早在HTTP建立之初， 主要是为了将超文本标记语言（HTML）文档从web服务器传送到客户端的浏览器， 也就是对于前端来说，我们所写的HTML页面是放在web服务器上的， 用户端通过浏览器访问url地址来获取网页的内容， 但是到了web2.0， 页面开始变的复杂， 不仅仅是简单的文字和图片， 同时HTML页面有了css, javascript, 并且随着ajax技术的出现， 又多了一种向服务端获取数据的方法， 但这些都是基于http协议的， 随着情况的更加复杂， 不得不对HTTP进行优化。



## 2. HTTP1.0和HTTP1.1的区别

1. **缓存策略**， 在HTTP1.0中主要是用header里的if-Modified-Since,Expires（到期）来作为缓存判断的标准， HTTP1.1则引入了更多的缓存控制策略，例如Entity tag（实体标签）,  if-Unmodified-Since, if-Match, if-None-Match等更多可控选择的缓存头来控制缓存策略
2. **带宽优化及网络连接的使用**， 在HTTP1.0中，存在一些浪费带宽的现象，比如说客户端只是需要某个对象的一部分，而服务器却将整个对象送过来了，并且不支持断点续传的功能， HTTP1.1则在请求头引入了range， 他允许只请求资源的某个部分，这样方便开发者自由的选择以便于充分利用带宽和连接
3. **新增部分错误通知**， 在HTTP1.1中新增了24个错误状态响应码， 比如说， 409请求到资源与当前的状态发生冲突， 410表示服务器上某个资源被永久性的删除
4. **Host头处理**，在HTTP1.0中认为每台服务器都绑定了一个唯一的IP地址，因此， 请求消息中的url并没有传递主机名， HTTP1.1中的请求消息和响应消息都应支持host头域，且请求消息中如果没有HOST头这个字段的话会报告一个400的错误
5. **长连接**， HTTP1.1支持长连接和请求的流水线处理，在一个TCP连接上可以传送多个HTTP请求和响应， 减少了建立和关闭连接的消耗和延迟， 在HTTP1.1中默认开启Connection: keep-alive, 一定程度上弥补了HTTP1.0每次请求都要创建连接的缺点



## 2. HTTP1.X和HTTP2.0的区别

1. **新的二进制格式**， HTTP1.X的解析是基于文本的，基于文本协议的格式解析存在天然缺陷，文本的表示形式有多样性，二进制则不同只认0和1的组合， 基于这种考虑HTTP2.0决定采用二进制格式，实现方便且健壮
2. **多路复用**，即连接共享，每一个request都是用做连接共享机制的，一个request对应着一个id， 这样一个连接上可以有多个request，每个连接的request可以随机的混杂在一起，接收方可以根据request的id将request再归属到各自不同的服务器端请求里面
3. **header压缩**，HTTP1.X中header带有大量的信息， 而且每一次都要重复发送，HTTP2.0使用encoder来减少需要传输的header大小
4. **服务端推送**， 可以通过解析HTML中的依赖， 返回其所需的其他文件（css, js）而不用再发送一次请求
