# 对vite的理解？   面试题

## 1.谈谈你对vite的理解，最好对比webpack说明

1. webpack会先从入口文件开始逐步分析依赖，进行打包，然后启动开发服务器，请求服务器时直接给的是打包后的结果



2. **而vite是直接开启开发服务器，请求到哪个模快再对该模快进行编译**，由于现代浏览器本身就支持ES Module, 会自动向依赖的module发出请求，vite充分利用这一点，将开发环境下的模快文件就作为浏览器要执行的文件， 而不像webpack那样进行打包合并



3. vite在启动的时候不需要打包，也就意味着不需要分析模快的依赖不需要编译，因此启动的时间非常快，当浏览器请求某个模快时，再根据需要对模快内容进行编译，这种**按需动态编译的方式，极大的缩减了编译的时间**，项目越复杂，模快越多，vite的优势就越明显



4. 当改动一个模快后，仅需让浏览器重新请求该模快即可，不像webpack那样需要把该模快的相关依赖模快全部编译一次， 所以vite的效率更高



5. 当需要打包到生产环境时，vite使用传统的rollup进行打包，因此，vite的主要优势在开发阶段，另外vite利用的是ES Module, 因此在代码中不可以使用CommonJs
