## webpack的编译过程分析

## 1. 初始化

1. 把输入的命令和配置文件和默认的配置进行统一，形成一个最终的配置对象
2. 对配置的处理过程是依托一个第三方库yarns完成的



## 2. 编译

1. 创建一个chunk(可以有多个chunk)，chunk就是一个块，他通过这个入口文件来梳理整个的依赖关系，这个chunk有两个属性， name: 默认为main，id: 如果是开发环境和name相同，如果是生产环境则是数字从0开始。

   ![chunk1](D:\CL武林秘籍\画图\chunk1.png)

2. 构建所有的依赖模块

```js
1. 先从入口文件分析，获取到他的文件路径，去和chunk中的模块记录表作比较，看这个路径是否被记录过，一开始记录表肯定是空的。

2. 如果记录表中存在这个模块路径，则直接结束，

3. 如果不存在，就读取文件，进行抽象语法树分析(AST), 记录依赖

4. 然后把记录下来的依赖保存到dependencies这个数组中，例如['./src/index.js', './src/test.js']

5. 然后替换依赖函数  require()  =>  _webpack_require()

6. 然后把转换的代码保存到这个模块记录表中

7. 在遍历这个dependencise数组，重复上述操作

8. 最后会形成一张总的模块记录表
```

![chunk2](D:\CL武林秘籍\画图\chunk2.png)

3. 根据这个模块记录表形成一个资源列表，格式如下

```js
文件名: ./dist/main.js
内容: 编译的结果代码

根据这些内容生成一个chunk hash字符串
```

![chunk3](D:\CL武林秘籍\画图\chunk3.png)

4. 将多个chunk资源列表合并到一起, 生成一个总的hash

![chunk4](D:\CL武林秘籍\画图\chunk4.png)

1. 根据node中的文件模块，把总的资源列表生成相应文件
